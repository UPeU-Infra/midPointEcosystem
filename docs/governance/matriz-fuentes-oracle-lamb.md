## Propósito

MidPoint (gestor de identidades, IGA) lee información de la base Oracle LAMB para construir las identidades digitales de la UPeU (estudiantes, trabajadores, docentes, egresados, áreas y puestos) y aprovisionar cuentas en sistemas destino (Microsoft 365 / Entra ID, Active Directory, Koha, LDAP/Keycloak, etc.).

Históricamente hemos tenido errores por **asumir** el nombre o significado de tablas/columnas (ejemplos reales: usar `NOMBRE_AREA` cuando la columna real es `NOMBRE`; usar `ID_SEDEAREA` cuando se necesitaba `ID_AREA`; faltar el JOIN a `TIPO_NIVEL_ENSENANZA` que dejaba el nivel de estudio vacío). Este documento existe para **no asumir**: lista EXACTAMENTE qué tablas y columnas estamos consumiendo hoy, para qué, y bajo qué supuesto. Pedimos a los dueños de la base que **confirmen, corrijan o señalen una mejor fuente** en la columna "Respuesta DBA".

> **NOTA IMPORTANTE — Oracle LAMB es SOLO LECTURA para MidPoint.** Todos los conectores tienen `create=false / update=false / delete=false`. MidPoint **JAMÁS** ejecuta INSERT / UPDATE / DELETE / DDL sobre LAMB. Solo hace SELECT.

## Huecos detectados (entidades SIN fuente Oracle clara en el repo)

Estas entidades del modelo IGA **no tienen** hoy un origen Oracle confirmado — son candidatas a "qué nos falta":

1. **Relación área ↔ puesto** (qué puestos pertenecen a cada área del MOF). Tenemos áreas (`ORG_AREA`) y puestos (`PLLA_PUESTO`) por separado, pero no la relación estructural entre ambos. (Ver G4.)
2. **Instituciones afines / partner-institution** (Colegio Unión, Clínica Good Hope, ISTAT, AGTU). Hoy se manejan con un overlay manual (anclaje por `ID_AREA` 97/695/8208/760 a campus en `org.xml`), no con una fuente Oracle que las identifique como entidades. (Ver G6.)
3. **Autoridades / designaciones temporales** (rector, decanos, directores con cargo de confianza por periodo). El cargo se infiere del puesto (`PLLA_PUESTO.NOMBRE` / `DOCENTE_AUTORIDAD`), pero no hay una fuente clara de "vigencia de la designación". (Ver G12.)
4. **Régimen docente completo** (`MOISES.PERSONA_ACAD_REGIMEN` está poco poblada, ~509 de ~8655). Falta fuente completa de TC/TP/DE. (Ver fila 42 / G12.)
5. **Ciclo del alumno** se infiere de `ACAD_PLAN_CURSO.CICLO` (ciclo del curso), no de un campo de "ciclo actual del alumno". Puede no existir un único ciclo por alumno. (Ver fila 9.)
