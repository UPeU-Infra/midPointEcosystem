Criterios de aceptación al LDAP:
✅
Nombres confirmados por RENIEC
Los nombres y apellidos del perfil coinciden con los datos del DNI validados por la API RENIEC
✅
Email institucional validado
La cuenta @upeu.edu.pe existe y está activa en Azure Entra ID (aprovisionada por MidPoint)
Fases de validación del padrón:
Fase 1 — AHORA
Matriculados 2026
LAMB ya usó API RENIEC al inscribir → se consideran RENIEC-confirmados por defecto.

Si tienen email activo en EntraID → pasan a active → LDAP.
Fase 2 — Con API RENIEC
Revalidar 2026 + histórico
Con acceso a la API RENIEC: reconfirmar nombres de 2026 y avanzar hacia atrás: 2025, 2024, 2023…

Requiere: gestión de acceso vía área de Desarrollo UPeU.
Fase N — Padrón completo
Completar padrón histórico
Todos los egresados, docentes y administrativos con registros históricos pasan por el pipeline de validación.

El Identity Cache acumula identidades curadas permanentemente.
Nota: El Active Directory existente de UPeU no se modifica. OpenLDAP es un directorio nuevo, independiente, dedicado exclusivamente a identidades curadas que han pasado el pipeline de calidad. Su propósito es proveer una fuente de identidades confiable para aplicaciones institucionales sin contaminar el AD productivo.
